Kothar

Kothar is a King of the Hill (KOTH) plugin with:

  • Multiple capture modes (including OVERLOAD multi-zone mode).
  • Built-in XP + Coins economy (database-backed).
  • Ranks progression system based on XP.
  • Leaderboards (/kothar top).
  • BossBar / Scoreboard / Hologram displays.
  • Shop GUI (/kothar shop) driven by configuration.
  • Discord webhook notifications.
  • Update checking/downloading from GitHub releases.

Requirements

  • Server: Spigot/Paper (recommended)
  • Dependency:
    • Apollo-Bukkit (required, hard-depend)

Optional Integrations (softdepend)

  • PlaceholderAPI
  • Vault
  • ProtocolLib
  • packetevents
  • TAB
  • Multiple team/clan plugins (ClansX, SimpleClans, UltimateClans, BetterTeams, Factions, etc.)

Quick Start

  1. Install Apollo-Bukkit.
  2. Put Kothar-<version>.jar into plugins/.
  3. Start the server.
  4. Configure:
    • plugins/Kothar/config.yml
    • plugins/Kothar/database.yml
  5. Reload:
    • /kothar reload

First KOTH Setup (Classic)

  1. Create the KOTH:
    • /kothar create <name>
  2. Select and save zone:
    • /kothar select <name>
    • Use the wand to select pos1 and pos2
    • Run /kothar select <name> again to save and exit
  3. Start:
    • /kothar start <name>

OVERLOAD Setup (4 Zones)

  1. Ensure KOTH is configured to OVERLOAD (via GUI/editor).
  2. Enter overload edit mode:
    • /kothar select <name>
    • This will instruct you about 4 zones.
  3. Select region and set each color:
    • /kothar setzone <name> RED
    • /kothar setzone <name> BLUE
    • /kothar setzone <name> GREEN
    • /kothar setzone <name> YELLOW

KOTH Setup

Classic KOTH

  1. /kothar create <name>
  2. /kothar select <name>
  3. Select pos1/pos2 with the wand
  4. /kothar select <name> again to save
  5. /kothar start <name>

OVERLOAD KOTH

Requires 4 zones: RED, BLUE, GREEN, YELLOW

  1. Put the KOTH into OVERLOAD mode (GUI/editor).
  2. /kothar select <name> (shows instructions)
  3. Select a region and apply:
    • /kothar setzone <name> RED
    • /kothar setzone <name> BLUE
    • /kothar setzone <name> GREEN
    • /kothar setzone <name> YELLOW

KOTH Files (koths folder)

Kothar stores each arena as a YAML file inside:

  • plugins/Kothar/koths/

Each file is named:

  • <kothName>.yml

After editing a file, run:

  • /kothar reload

Economy (XP & Coins)

Kothar includes a built-in economy:

  • XP (used for ranks)
  • Coins (used for shop & player trading)

Rewards

Configured in config.yml:

  • Kill rewards
  • Win rewards
  • Capture tick rewards
  • Death penalties

Player trading

  • /kothar pay <player> <amount>

Ranks

Ranks are defined in ranks.yml. Players progress based on XP.

Commands

  • /kothar rank

Rank rewards

Use rewards commands in ranks.yml. Use {player} placeholder.

Troubleshooting

“This command can only be executed by a player”

You ran a player-only command from console. Use in-game.

“Not enough players online…”

Your KOTH has min-players > online players.

“KOTH is not properly configured”

You must configure the zone/spawn/settings via GUI or selection.

Shop purchases do nothing

  • Check console for command errors from shops.yml.
  • Validate material names and command syntax.

Update command returns 404 / fails

  • Verify repo settings and that a release exists.
  • If private repo: set GITHUB_TOKEN.

Coins command fails for offline players

If you run an offline-mode server, UUID resolution may differ from Mojang UUIDs.
